# 拾光群像展示页｜第一版范围卡

## 1｜文件定位

本文件用于定义「拾光群像」网站第一版的制作范围。

它不是完整网站策划案，也不是内部人物资产说明书。  
它用于帮助 Codex 和后续开发过程明确：

- 第一版网站做什么；
- 第一版网站不做什么；
- 面向谁展示；
- 展示哪些人物；
- 每个人物如何用大众语言介绍；
- 代表作品如何展示；
- 图片、试听片段、视频暂时如何占位。

## 2｜网站定位

「拾光群像」是一个面向大众的虚拟人物展示网站。

它用于介绍一组以音乐、影像和故事逐渐生长的虚拟人物。

网站第一版的核心不是展示内部人物资产，也不是展示音色模型、生成规则或技术参数。

它要让普通访问者知道：

1. 拾光群像是什么；
2. 目前有哪些虚拟人物；
3. 每个人物大致是什么气质；
4. 每个人物通常唱什么类型的歌；
5. 每个人物有哪些代表作品；
6. 后续可以在哪里听到或看到相关内容。

## 3｜网站名称与副标题

### 网站名称

```text
拾光群像
```

### 建议副标题

```text
在声音与故事里相遇的人
```

### 网站性质

```text
虚拟人物音乐企划展示页
```

## 4｜目标访问者

第一版网站面向：

- 普通听众；
- 对虚拟人物感兴趣的人；
- 从歌曲、朋友圈、视频或分享链接进入的人；
- 想初步认识这些人物的人。

第一版网站不面向：

- 内部创作管理；
- 音色模型调用；
- Suno 生成管理；
- 技术展示；
- 人物资产归档。

## 5｜第一版人物范围

第一版只展示四位人物：

1. 苏晚诗
2. 苏晴栀
3. 江弈辰
4. 许念安

第一版暂不展示：

```text
乔曼宁
```

乔曼宁可作为后续扩展人物，不进入第一版页面范围。

## 6｜第一版页面目标

第一版只解决四件事：

1. 说明「拾光群像」是什么；
2. 展示四位虚拟人物；
3. 用大众化语言介绍每个人物的气质、故事感和常见歌曲类型；
4. 预留代表作品、图片、试听片段和视频入口。

## 7｜展示语言原则

对外页面不使用内部模型语言。

避免使用：

- 音色模型
- 适配题材
- 不适配方向
- 调用边界
- 相似度评估
- 生成提示词
- Suno 参数
- 资产管理
- 内部正本

统一改为大众可理解的表达：

- 她 / 他是谁
- 她 / 他给人的感觉
- 她 / 他常唱什么样的歌
- 她 / 他的代表作品
- 可以在哪里听到 / 看到她或他

## 8｜第一版页面结构

建议页面顺序：

1. Hero：首页主视觉
2. About：关于拾光群像
3. Characters：四位人物总览
4. Character Detail：人物详情
5. Works：人物详情中的代表作品入口
6. Media：人物详情中的图片 / 试听片段 / 视频展示占位
7. Footer：项目说明与后续入口

第一版建议使用单页结构。  
人物卡片可以通过锚点跳转到对应人物详情区。  
暂不需要复杂多页面路由。

代表作品和媒体占位建议放在每个人物详情中，不必单独做很大的全站 Works 区或 Media 区，避免第一版页面显得空。

## 9｜首页 Hero 文案建议

```text
拾光群像

在声音与故事里相遇的人。

这里收录一组正在成长的虚拟人物。
他们有各自的声音、性格、故事和歌曲。
有的人属于夜色，有的人属于日常光线；
有的人把人带向远处，有的人把灯留到很晚。
```

## 10｜关于拾光群像文案建议

```text
拾光群像是一组以音乐、影像和故事逐渐生长的虚拟人物。

他们不是一次性的角色设定，而是在不同作品中持续出现、被慢慢认识的人。

每个人都有自己的声音气质、情绪底色和适合讲述的故事。
```

## 11｜人物展示内容

每个人物详情建议包含：

1. 人物姓名；
2. 一句话定位；
3. 人物介绍；
4. 常唱歌曲类型；
5. 代表作品；
6. 图片展示占位；
7. 试听片段占位；
8. 视频展示占位。

不展示：

- 内部音色模型全文；
- 适配 / 不适配判断；
- 生成参数；
- 反向提示词；
- 内部人物一致性规则。

## 12｜四位人物文案

### 12.1｜苏晚诗

#### 一句话定位

```text
在夜色和回忆里低声讲故事的人。
```

#### 人物介绍

```text
苏晚诗的歌，常常发生在夜晚、雨声、空房间和城市的灯影里。

她不是把情绪大声唱出来的人，而是把没有说完的话，留在一句歌词的尾音里。

她的作品多半安静、克制，带一点遗憾，适合在深夜一个人慢慢听。
```

#### 常唱歌曲类型

```text
夜色情歌
都市遗憾
慢速抒情
回忆感作品
带冷感的电子流行
```

#### 代表作品

```text
《纱花之约》
《霓虹梦境》
```

#### 页面氛围

```text
夜色、雨、窗、灯影、冷灰、暗蓝、低饱和、安静留白。
```

---

### 12.2｜苏晴栀

#### 一句话定位

```text
把普通生活唱出柔亮微光的人。
```

#### 人物介绍

```text
苏晴栀更像日常光线里轻轻靠近的人。

她的歌常有花、风、咖啡、街角和明亮的小心情。

她可以温柔，也可以轻快；可以有一点梦感，但不会离生活太远。

听她的歌，像在普通的一天里，忽然看见一点柔亮的光。
```

#### 常唱歌曲类型

```text
温柔流行
轻梦感抒情
生活感小歌
轻甜但不幼态的情歌
明亮治愈作品
```

#### 代表作品

```text
《迷蝶》
《苏晴栀 Iris》
```

#### 页面氛围

```text
日光、花、窗、咖啡、街角、浅色、柔亮、清透、生活感。
```

---

### 12.3｜江弈辰

#### 一句话定位

```text
温暖、明亮、让人愿意一起往前走的人。
```

#### 人物介绍

```text
江弈辰的声音里有一种可靠感。

他适合唱那些关于陪伴、行动、团结、出发和继续向前的歌。

有时他像站在人群前带大家一起唱，有时也会安静下来，认真讲自己的故事。

他的歌通常不沉重，而是带着一种明亮、松弛、愿意继续生活的力量。
```

#### 常唱歌曲类型

```text
正向流行
主题歌
陪伴感歌曲
轻雷鬼 / 轻律动
带行动感的男声作品
```

#### 代表作品

```text
《爱与团结》
《睡不着有限公司》
```

#### 页面氛围

```text
阳光、街头、海边、风、路、舞台、暖亮、开阔、行动感。
```

---

### 12.4｜许念安

#### 一句话定位

```text
在深夜把灯留到很晚的治愈播主。
```

#### 人物介绍

```text
许念安像一个深夜还在麦克风前的人。

他的歌不急着把谁拯救，也不说太多大道理。

更多时候，他只是把声音放低，把灯留着，陪人把一天慢慢放下。

他的作品常有暖灯、书房、海边、旧时光和睡前的安静感。
```

#### 常唱歌曲类型

```text
深夜治愈
睡前陪伴
温柔男声慢歌
轻民谣 / 轻流行
旧时光与人间归处感作品
```

#### 代表作品

```text
《灯影归心》
《人间归舟》
```

#### 页面氛围

```text
暖灯、书房、麦克风、耳机、木桌、吉他、海边黄昏、低饱和暖色。
```

## 13｜代表作品展示规则

第一版每人展示两首代表作。

代表作品第一版只显示名称，不要求提供播放链接。

如果没有真实链接，不要创建空链接或 404 链接。

可以显示：

```text
作品入口即将上线
```

或：

```text
敬请期待
```

## 14｜素材占位规则

第一版不要求提供真实图片、试听片段、视频和作品链接。

代表作品、图片、试听片段、视频展示均先做成安全占位模块。

要求：

1. 不要使用会报错的空媒体文件；
2. 不要创建点击后 404 的无效链接；
3. 如果没有真实素材，显示“即将上线 / 敬请期待 / 内容准备中”；
4. 页面结构要预留未来接入真实图片、试听片段、视频和作品链接的位置；
5. 人物数据中可以预留 image、audio、video、works 字段；
6. 字段为空时页面应正常显示占位状态；
7. 占位模块要像设计好的“即将上线”，不要像素材缺失或页面没做完；
8. 后续补充真实素材后，可以通过修改人物数据文件完成替换，不需要重做页面结构。

## 15｜第一版不做事项

第一版不做：

- 登录；
- 后台；
- 数据库；
- 评论区；
- 会员系统；
- 完整音乐库；
- 完整视频库；
- 复杂人物关系系统；
- 内部模型正本展示；
- 音色生成规则展示；
- 适配 / 不适配判断展示；
- 技术参数展示。

## 16｜视觉风格

整体视觉关键词：

```text
电影感
低饱和
文艺
干净
安静
现代
有音乐项目感
有虚拟人物感
但不强二次元
不企业官网
不技术展示页
```

更准确的视觉方向：

```text
像一个虚拟人物音乐企划的官方展示页，而不是开发者作品集。
```

## 17｜数据结构建议

人物内容建议统一放在独立数据文件中，例如：

```text
src/data/characters.js
```

或：

```text
src/data/characters.ts
```

建议字段：

```text
id
name
tagline
intro
songTypes
works
visualMood
image
audio
video
```

其中：

```text
image
audio
video
```

第一版可以为空。

页面检测到为空时，应显示占位状态，而不是报错。

## 18｜第一版成功标准

第一版成功标准：

1. 网站能公开访问；
2. 访问者能知道「拾光群像」是什么；
3. 四位人物气质清楚；
4. 页面不像技术文档，而像一个虚拟人物音乐企划展示页；
5. 人物介绍大众能看懂；
6. 没有暴露内部音色模型、生成参数或资产管理语言；
7. 没有无效链接、报错播放器或 404 占位；
8. 后续可以继续扩展图片、歌曲、视频和更多人物。

## 19｜第一版核心原则

```text
先让大众认识人物
再逐步补充作品入口
最后再扩展更多媒体内容
```

第一版不追求功能多，而追求人物清楚、气质准确、页面稳定、未来可扩展。
